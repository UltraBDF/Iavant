# 04 - Conventions d'Ingénierie

## 1. Style de Code

- **TypeScript :** Mode strict. Pas de `any`. Interfaces pour les props.
- **Composants :** Noms en PascalCase. Composants fonctionnels uniquement.
- **Dossiers :** Organisation par fonctionnalité dans `src/app`.

## 2. Styling (Tailwind 4)

- Utilisation des variables CSS pour N.E.O.
- Pas de couleurs "hard-coded" pour supporter le mode contraste élevé.

## 3. Workflow Git

- **Branches :** `feat/`, `fix/`, `docs/`.
- **Commits :** Conventional Commits (`feat:`, `fix:`, etc.).
- **PR :** Description claire et capture d'écran obligatoire si visuel.
